## 5. Internal models and invariants

`YogaDefinitionDisposition` is exactly `valid` and `invalid`.

`YogaEvaluationRecord` fields are:

1. `yoga_id`;
2. `name`;
3. `rule_version` (the current validated source value, integer `1` today);
4. `source` (`RuleSourceIdentity`, including logical source name, rule ID, and zero-based index);
5. `definition_disposition`;
6. ordered `definition_issues`;
7. complete canonical `condition_result`;
8. Yoga `matched`;
9. typed `status`;
10. deterministic `trace_reference`;
11. immutable `compatibility_evidence` needed to reproduce the characterized lossy shape;
12. immutable `compatibility_houses`;
13. optional `evaluation_time_ms`.

`source_index` is a read-only property derived from `source.rule_index`. A valid record has no definition issues and exactly one result. An invalid record has one or more WP12 issues and may have the locked safe runtime result. `matched` is true exactly for `PredicateStatus.MATCHED`; any retained condition result must agree. UUID references must be version 5. Nested result/evidence/issues are immutable and detached. Equality excludes the record duration and recursively uses WP02/WP10 telemetry-neutral result equality.

`YogaEvaluationBatch` fields are `schema_version`, `evaluator_version`, `prepared_state_digest`, source-ordered `records`, typed `batch_issues`, and optional `total_duration_ms`. Record indices must be exactly zero-based source order. Batch equality excludes total duration. No batch or record retains AstroState, a provider, callable, evaluator/cache, source path, raw rule mapping, or compatibility row dictionary.

`YogaRuleSource` is the explicit immutable source/validation carrier. `YogaLegacyPreparation` holds the existing WP07 `PreparationOutcome` plus an immutable compatibility-only diagnostic graph used solely to reconstruct the already-public legacy evidence.

## 6. Serialization and identity

Logical Yoga bytes include schema/evaluator versions, prepared digest, rule identity/name/version/source, disposition and exact WP12 issues, complete recursive condition logical projection, Yoga matched/status, deterministic reference, and compatibility evidence/houses. They exclude all Yoga/condition/predicate durations and recursive leaf cache-hit telemetry.

Full bytes add Yoga durations and use WP03 full recursive results, so cache-hit telemetry remains observable. The active evaluator deliberately stores optional wall-duration fields as `None`; this retains all logical content and cache telemetry while making equivalent cold-process full bytes reproducible. Cold/warm full bytes differ only where leaf `cache_hit` differs. Logical bytes/equality/hash and public compatibility never differ with warmth.

Canonical JSON is compact UTF-8 with deterministic mapping order. Readers reject invalid JSON, duplicate keys, non-finite constants, unknown batch shape, and invalid nested canonical models. Hashes are lowercase SHA-256.
