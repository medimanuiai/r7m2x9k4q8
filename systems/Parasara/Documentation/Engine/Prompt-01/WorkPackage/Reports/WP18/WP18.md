## 9. Bounded performance method

No approved pre-Prompt-01 benchmark exists with the same workload, method, and
environment. `tests/testing_framework/perf.py` is not a valid baseline: it
benchmarks disallowed snapshot generation, writes the repository-root `-`, uses
a different method, and has no `tests/perf_baseline.json`. Consequently, the
following is descriptive evidence only. It makes no faster/slower/no-regression
claim and applies no post-hoc threshold.

The disposable in-memory harness:

- prebuilt fixture charts and immutable inputs before timed sections;
- excluded dependency imports, environment creation, pytest startup, network,
  raw Surya computation, snapshot work, and artifact file writes;
- used only `time.perf_counter_ns`;
- performed three fixed untimed warm-up operations per workload;
- ran seven independent samples in the same order in both lanes;
- used the iterations/sample shown below;
- timed only the inner operation loop, then consumed the last output and
  verified its exact SHA-256 outside the timed section;
- recreated evaluators/caches inside cold operations and retained explicitly
  prewarmed evaluators for warm operations;
- built WP16 tooling/coverage projections in memory;
- replaced only the manifest's file-writing tooling row with its in-memory
  logical equivalent, so `wp17_manifest_in_memory` includes no artifact writes
  or subprocess startup;
- reported microseconds per complete named operation. For example,
