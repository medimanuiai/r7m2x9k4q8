## 4. Exact model inventory and invariants

### Enumerations

| Enum | Exact values |
|---|---|
| `ConditionOperator` | `AND`, `OR`, `NOT` |
| `ConditionNodeDisposition` | `evaluated`, `skipped` |

### `ConditionChildResult`

Exact fields, in order:

1. `node_id`
2. `child_index`
3. `disposition`
4. `result`
5. `skip_reason`

An evaluated child has exactly one canonical `PredicateResult` or nested `ConditionResult` and no skip reason. A skipped child has no result and exactly one locked reason: `and_short_circuit_unmatched` or `or_short_circuit_matched`. Indexes are strict non-Boolean nonnegative integers; parent tuples enforce zero-based declared order and unique child node IDs.

### `ConditionResult`

Exact fields, in order:

1. `node_id`
2. `operator`
3. `matched`
4. `status`
5. `details`
6. `children`
7. `errors`
8. `trace_steps`
9. `evaluation_time_ms`

`matched` is a strict Boolean and is true exactly when status is `matched`. Operator and status are typed enums. Details are defensively deep-frozen and must identify the exact operator and a nonnegative declared child count. Children, errors, and trace steps are immutable typed tuples. `error` requires an error; factual matched/unmatched statuses reject errors. Duration is optional, finite, numeric, non-Boolean, and nonnegative.

Caller-owned mappings/lists are recursively copied into `FrozenMapping`/tuples. Cycles, non-string mapping keys, arbitrary objects, open resources, callables, non-finite floats, and other WP03-unsafe values fail explicitly. Logical equality excludes this node's duration and recursively relies on child logical equality, thereby also excluding every descendant predicate `cache_hit` and duration.

## 5. Runtime grammar and safety limits

Accepted bare current-format nodes only:

```text
logical: {"type": "AND"|"OR"|"NOT", "children": [...]}
leaf:    {"type": <registered predicate ID>, "params": {...}}
```

Type values use existing trim/uppercase normalization. Keys are not normalized and no aliases are accepted.

Rejected safely: null/non-mapping evaluated nodes, missing/blank/non-string/unknown types, unknown operators or definitions, unknown fields, `op/args`, `predicate_id`, `parameters`, missing/null/non-mapping params, leaf-with-children, logical-with-params, missing/null/non-list children, empty AND/OR, NOT arity other than one, cycles, depth 65, and node 4097. Depth is counted with the root as level 1; total node count includes the root. Depth 64 and node count 4096 are accepted.

A tree-wide preflight checks only cycle/depth/node safety. Node-local grammar and leaf semantics are evaluated on demand. Consequently, a short-circuited subtree receives a skipped representation without parameter validation or execution; only the permitted tree-wide safety walk can observe its structural links.

Recognized malformed logical nodes return a typed error `ConditionResult`. Inputs that cannot identify a valid logical operator or leaf return a safe `PredicateResult` with identity `CONDITION_BOUNDARY`; they are never mislabeled `AND`, `OR`, or `NOT` and never become factual unmatched.

## 6. Root-leaf policy

A valid root leaf returns the canonical WP02 `PredicateResult` directly. No `AND` wrapper, fake logical predicate ID, or `LEAF` operator was added. A leaf inside a logical node is retained intact in its `ConditionChildResult`.

Known but unmigrated predicates reach WP09 and return its typed `predicate_not_migrated` result. The canonical condition evaluator never calls the legacy evaluator or Yoga-local helpers.

## 7. Exact operator and precedence policies

All operators evaluate declared children left-to-right.

### AND

| Observed sequence | Parent result | Remaining children |
|---|---|---|
| all matched | matched | none |
| first unmatched with no earlier non-factual child | unmatched | skipped |
| non-factual child, later unmatched | highest prior non-factual status | skipped after unmatched |
| no unmatched, one or more non-factual children | highest non-factual status | none |

Non-factual children do not decide false and evaluation continues until an unmatched child or the end. Short-circuit reason is exactly `and_short_circuit_unmatched`.

### OR

| Observed sequence | Parent result | Remaining children |
|---|---|---|
| first matched, including after earlier non-factual children | matched | skipped |
| all unmatched | unmatched | none |
| no match and one or more non-factual children | highest non-factual status | none |

Short-circuit reason is exactly `or_short_circuit_matched`. Earlier non-factual children remain intact even though matched is decisive.

### NOT

| Child | Parent |
|---|---|
| matched | unmatched |
| unmatched | matched |
| error | error |
| timeout | timeout |
| invalid_parameters | invalid_parameters |
| missing_capability | missing_capability |
| skipped | skipped |

NOT requires exactly one child and never discards its evidence, errors, or trace.

### Mixed non-factual precedence

Highest first, locked by tests:

1. `error`
2. `timeout`
3. `invalid_parameters`
4. `missing_capability`
5. `skipped`

For equal statuses, the first evaluated child is the precedence source. The specific WP10 rule that AND continues past a non-factual child to seek a factual unmatched child resolves the older decision record's less-specific phrase “false/non-success”; the approved status precedence and no-false-fact invariants are preserved.

## 8. Skips, IDs, traces, errors, and evidence

Default IDs are `condition.root`, `condition.root.children.0`, and recursive extensions. Caller root IDs are validated stable path identifiers; no UUID, time, object ID, Python hash, or iteration-dependent value is used.

Each valid logical result contains deterministic parent-owned trace steps for:

1. operator start/validation;
2. each evaluated child result in order;
3. short-circuit decision when present;
4. each skipped child in declared order;
5. precedence selection when present;
6. final disposition.

Child traces remain inside child results and are not flattened. Parent steps reference only child indexes/IDs and stable statuses/reasons. They contain no raw subtree, provider value, timing, cache state, path, exception string, or traceback.

The recursive child tree is the evidence container; no aggregate factual evidence is synthesized. A non-factual logical parent owns one stable `PredicateError`. Its details contain only selected child ID/status and ordered child error codes. Raw child details remain solely in the child tree.
