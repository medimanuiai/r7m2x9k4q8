## 2. Before/after architecture

Before WP09, `rules/engine.py` owned `_CACHE`, a process-global mutable dictionary keyed by `(id(astro), uppercased name, json/default-str parameters)`. It cached every legacy result, including failures, shared entries across requests, exposed warm telemetry inside the stored value, and required Yoga/tests to clear global state.

After WP09:

```text
caller-owned PreparedAstroState + PredicateEvaluationContext
    -> PredicateEvaluator instance
    -> WP04 definition + WP05 validation
    -> PredicateCacheKey over definition/parameters/prepared content
    -> instance-owned synchronized bounded LRU
    -> WP08 evaluate_planet_in_house on a miss
    -> telemetry-free immutable stored result
    -> fresh cold/warm telemetry-decorated response
```

The legacy mutable-state `evaluate_predicate` remains compatibility-only and is now uncached. It does not enter or contaminate the canonical cache. `clear_cache()` remains a documented no-op symbol for active Yoga/test callers pending WP16 removal.

## 4. Public canonical APIs

Location: `systems/Parasara/engine/rules/evaluator.py`.

```python
class PredicateEvaluator:
    def __init__(self, *, capacity: int = 256) -> None: ...
    def evaluate(
        self,
        predicate_id: Any,
        parameters: Any,
        state: PreparedAstroState,
        context: PredicateEvaluationContext,
        *,
        use_cache: bool = True,
    ) -> PredicateResult: ...
    def clear_cache(self) -> None: ...
    def freeze_cache(self) -> None: ...

class PredicateResultCache:
    def __init__(self, *, capacity: int = 256) -> None: ...
    def get(self, key: PredicateCacheKey) -> PredicateResult | None: ...
    def peek(self, key: PredicateCacheKey) -> PredicateResult | None: ...
    def put(self, key: PredicateCacheKey, result: PredicateResult) -> bool: ...
    def clear(self) -> None: ...
    def freeze(self) -> None: ...
    def keys(self) -> tuple[PredicateCacheKey, ...]: ...

def build_predicate_cache_key(
    definition: PredicateDefinition,
    normalized_parameters: Mapping[str, Any],
    state: PreparedAstroState,
    *,
    relevant_context: Mapping[str, Any],
    context_relevance: str = "none",
) -> PredicateCacheKey: ...
```

`rules.engine.create_predicate_evaluator(capacity=256)` is a lazy compatibility-facing factory that avoids a process-global canonical evaluator.

## 5. Cache ownership, bound, LRU, lifecycle, and synchronization

| Concern | Implemented policy |
|---|---|
| ownership | one `PredicateResultCache` per `PredicateEvaluator` instance |
| default bound | exactly 256 entries |
| configurable bound | positive non-Boolean integer only |
| eviction | deterministic LRU; ordinary hit promotes to MRU |
| replacement | same key replaces in place without size growth and becomes MRU |
| synchronization | standard-library `RLock` around get/peek/put/clear/freeze/keys/size/frozen operations |
| handler lock boundary | handler executes outside cache lock; duplicate miss computation is allowed |
| clear | empties deterministically and is idempotent |
| freeze | idempotent; prevents insertion, replacement, eviction, and hit-recency mutation |
| frozen hits | remain readable |
| frozen misses | evaluate cold and are not inserted |
| clear after freeze | allowed as a one-way data-reduction operation; frozen state remains frozen |
| exposure | no mutable entry/order mapping is returned; key enumeration is an immutable tuple |

Concurrent stress interleaved evaluation, insertion, freeze, and clear across eight workers. All responses remained canonical, completed operation size never exceeded three, keys remained unique, and no structure/value corruption occurred.

## 8. Stored value and response telemetry

The cache accepts only WP02 canonical `PredicateResult` values. Before storage it creates an immutable telemetry-free value with `cache_hit=False` and `evaluation_time_ms=None`.

On a cold evaluation, the evaluator returns a fresh result with `cache_hit=False` and a finite nonnegative duration. On a hit, it returns a fresh result with `cache_hit=True` and a new finite nonnegative duration. It never returns a telemetry-decorated stored object.

Cold and warm results have identical predicate ID/version, inputs, evidence, trace, errors, status, matched value, logical projection, logical bytes, and logical SHA-256. Only WP03 full telemetry projection differs. A controlled clock test proves the cold duration is not reused for the hit.

`use_cache=False` bypasses get/put, returns cold telemetry, and does not change storage or LRU order.

## 9. Status and recovery matrix

| Status | Cached | Recovery behavior |
|---|---:|---|
| `matched` | yes | warm read permitted |
| `unmatched` | yes | warm read permitted |
| `missing_capability` | no | every request re-executes |
| `invalid_parameters` | no | key construction is skipped |
| `error` | no | every request re-executes |
| `timeout` | no | cache rejects controlled value |
| `skipped` | no | cache rejects controlled value |

Missing entity and missing fact are `missing_capability` and do not consume capacity. A controlled error followed by a repaired handler result under the same normalized key executes twice and succeeds without clearing. Cache storage does not rewrite result status, evidence, trace, or errors.

## 10. Canonical routing and compatibility boundary

The evaluator resolves the WP04 definition, validates with WP05 before key construction, canonicalizes alias identity, checks the typed WP07 state/context and scope, and routes only `PLANET_IN_HOUSE` through WP08 `evaluate_planet_in_house`. Factual/capability logic is not duplicated.

Unknown requests return canonical typed `status=error`, code `unknown_predicate`. Known but unmigrated requests return code `predicate_not_migrated`. Mutable `AstroState` returns `canonical_boundary_type_mismatch`. None is cached. `ASPECT` and `ASPECT_EXISTS` resolve to the same canonical unmigrated identity and create no duplicate entries.

The six-ID/five-handler registry remains unchanged. `HOUSE_OCCUPANT`, `ASPECT`, `ASPECT_EXISTS`, `FUNCTIONAL_ROLE`, and `PLANET_EXALTED` retain legacy execution only. Legacy `evaluate_predicate` is deliberately uncached; its prior process-global warm assertion and fixtures were updated solely for this intentional ownership change. `clear_cache()` is a no-op compatibility symbol planned for WP16 removal.
