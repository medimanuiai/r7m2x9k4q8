## 3. Supported and rejected canonical values

| Category | Policy |
|---|---|
| `None` | supported unchanged |
| actual Boolean | supported; never coerced to integer |
| integer | supported; Boolean excluded |
| finite float | supported; `-0.0` normalized to `0.0`; NaN/infinities rejected |
| string | preserved exactly; no case or Unicode normalization |
| string-valued enum | normalized to its declared string value |
| mapping | actual string keys only; values recursively canonicalized |
| list/tuple | both frozen to ordered tuples and projected as JSON arrays |
| explicit predicate models | supported only through dedicated projection functions |
| set/frozenset | rejected as unordered producer data |
| non-string mapping key | rejected at its logical mapping path |
| binary values | bytes, bytearray, and memoryview rejected |
| numeric/time identifiers | Decimal, complex, date/time/timedelta, and UUID rejected |
| environment/runtime values | paths, regex, modules, classes, callables, generators, iterators, exceptions, tracebacks, resources, locks, and sockets rejected |
| framework/custom objects | arbitrary dataclasses, Pydantic models, and custom objects rejected |
| non-string enum | rejected |
| cyclic data | rejected at the first active-ancestry recurrence |

Errors contain a safe logical path and stable value category. They do not include the value, raw exception text, memory address, `repr`, or secret payload. Harmless repeated references are accepted and independently frozen because cycle detection tracks only active recursion ancestry.

## 4. `FrozenMapping` and equality policy

`FrozenMapping` provides immutable lookup, membership, length, deterministic exact-key lexicographic iteration, safe length-only representation, logical insertion-order-independent equality, and process-local hashability. It exposes only a tuple and `MappingProxyType`, so no mutable backing dictionary escapes. Construction recursively copies all supported caller values.

Canonical equality is type-sensitive. In particular, integer `1` and float `1.0` are not logical equals because their strict JSON bytes differ. Mapping and tuple equality recurses under the same policy. Python `hash()` is consistent with this equality but is not persisted or compared across processes; persisted identity is SHA-256 over canonical JSON bytes.

## 5. Exact projection schemas

`PredicateError` data contains exactly:

```text
code, message, predicate_id, details, recoverable
```

`PredicateTraceStep` data contains exactly:

```text
step_id, operation, details, observation, parent_step_id, error_code
```

Optional trace values remain explicit JSON `null`.

Logical `PredicateResult` data contains exactly:

```text
matched, predicate_id, predicate_version, inputs, evidence,
trace_steps, errors, status
```

Full diagnostic `PredicateResult` data contains the same eight fields plus:

```text
cache_hit, evaluation_time_ms
```

No wrapper, schema envelope, class/module name, timestamp, or hidden metadata was added. Tuples project as arrays and frozen mappings as objects. Empty mappings/tuples remain explicit `{}` and `[]`.

## 6. Canonical JSON policy

All canonical JSON uses one configuration:

- UTF-8 without BOM;
- `ensure_ascii=False`;
- `allow_nan=False`;
- exact lexicographic object-key sorting at every level;
- separators `(',', ':')`;
- no indentation, trailing whitespace, or platform newline;
- lowercase JSON literals from the standard encoder;
- no `default` callback or coercive fallback.

The API provides general canonical value data/text/bytes functions plus dedicated error, trace, logical-result, and full-result functions. Representative nested UTF-8 byte literals are asserted exactly in tests.

## 7. Round-trip policy

Dedicated deserializers exist for `PredicateError`, `PredicateTraceStep`, logical `PredicateResult`, and full `PredicateResult`, with explicit text/bytes entry points.

They:

- accept only JSON text or strict UTF-8 bytes at JSON boundaries;
- reject duplicate JSON keys;
- reject malformed JSON/UTF-8 and non-finite numeric input;
- require exact field sets and reject missing/unknown keys;
- reject incorrect containers and element types;
- reconstruct only the three known canonical model types;
- reapply all WP02 constructor invariants and canonical value validation;
- reject unknown status values;
- restore logical-result telemetry to `cache_hit=False` and `evaluation_time_ms=None`;
- preserve full-result telemetry exactly.

No arbitrary dataclass/Pydantic reflection or tagged-type instantiation is used.

## 8. Logical hash policy

`predicate_result_logical_sha256(result)` hashes the exact canonical logical UTF-8 bytes and returns lowercase 64-character SHA-256. It excludes cache and duration telemetry, is stable across calls/processes/Python lanes, and changes for matched/status, predicate identity/version, inputs, evidence, traces, or errors.

Canonical models remain deliberately unhashable. The digest does not use Python `hash()`, object identity, `repr`, or pickle.
