## 3. Canonical model contract

| Model | Field | Type/shape | Construction rule |
|---|---|---|---|
| `PredicateStatus` | enum value | string enum | exactly `matched`, `unmatched`, `missing_capability`, `invalid_parameters`, `error`, `timeout`, `skipped` |
| `PredicateError` | `code` | `str` | non-empty, no coercion |
| | `message` | `str` | non-empty safe model data |
| | `predicate_id` | `str` | non-empty, no coercion |
| | `details` | immutable mapping | required, recursively defensively frozen; unsafe runtime objects rejected |
| | `recoverable` | `bool` | actual Boolean only |
| `PredicateTraceStep` | `step_id` | `str` | required non-empty stable caller-supplied path ID |
| | `operation` | `str` | required non-empty stable operation name |
| | `details` | immutable mapping | required and recursively frozen |
| | `observation` | immutable value/mapping | recursively frozen; never generated from time/randomness |
| | `parent_step_id` | `str \| None` | optional; non-empty when present |
| | `error_code` | `str \| None` | optional; non-empty when present |
| `PredicateResult` | `matched` | `bool` | actual Boolean only |
| | `predicate_id` | `str` | required non-empty identity |
| | `predicate_version` | `str` | required non-empty version; WP02 does not impose SemVer parsing |
| | `inputs` | immutable mapping | independent empty default; `None` rejected |
| | `evidence` | immutable mapping | independent empty default; `None` rejected |
| | `trace_steps` | `tuple[PredicateTraceStep, ...]` | typed tuple preserving semantic order |
| | `errors` | `tuple[PredicateError, ...]` | typed tuple preserving semantic order |
| | `cache_hit` | `bool` | actual Boolean; defaults to false; telemetry |
| | `evaluation_time_ms` | finite nonnegative numeric or `None` | Boolean, negative, NaN, and infinity rejected; telemetry |
| | `status` | `PredicateStatus` | mandatory typed status |

Truth invariants enforced:

- `matched=True` if and only if status is `matched`;
- every other status requires `matched=False`;
- `error` requires at least one typed error;
- ordinary `matched` and `unmatched` factual outcomes cannot carry errors;
- trace/error collections accept only their canonical typed elements;
- required collections reject `None`.

The trace field names reconcile the locked plan’s descriptive wording (“inputs/details” and “result or observation”) with the WP02 prompt’s exact field contract: the implemented names are `details` and `observation`. No speculative operation enum or logical telemetry field was added.

## 4. Immutability and equality decisions

All three dataclass models are frozen. The private `_ImmutableMapping` and `_freeze_value` support in `models.py` recursively copy/freeze mappings, lists/tuples, and sets; reject cycles and unsupported mutable/runtime objects; and leave no caller-owned mutable object reachable through constructed state. Raw exceptions, traceback objects, callables, and open I/O resources are rejected from nested model data.

This helper is private and deliberately minimal. It is not the project-level `FrozenMapping`, canonical ordering, supported-value, serializer, round-trip, or hashing API.

`PredicateResult.__eq__` compares only logical fields:

- matched value;
- predicate identity and version;
- inputs and evidence;
- trace steps and errors;
- status.

`cache_hit` and `evaluation_time_ms` do not affect equality. Equality is symmetric and transitive and remains stable after original caller inputs are mutated. Every canonical model is deliberately unhashable. WP03 owns canonical logical projection and any corresponding hash implementation.

## 5. WP02/WP03 boundary

WP02 implemented only construction validation, private defensive freezing, immutable ownership, and logical equality. It did not implement:

- a public `FrozenMapping` or general freeze API;
- canonical key ordering;
- logical/full JSON projections;
- serializers or deserializers;
- JSON byte stability or round trips;
- reusable canonical hashing;
- evaluator error conversion or an error-code catalog;
- handler/evaluator/caller migration.

Those serialization/freeze responsibilities remain in WP03. The private helper is explicitly marked for consolidation/replacement there.
